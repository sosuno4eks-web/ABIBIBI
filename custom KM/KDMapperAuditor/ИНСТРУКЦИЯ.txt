ЧТО ПРОВЕРЯЕТ
-------------

[1] String Scan          → Сканирует память на подозрительные строки
[2] MmUnloadedDrivers    → Проверяет список выгруженных драйверов
[3] PiDDBCache           → Проверяет базу данных драйверов
[4] Event Log Scan       → Сканирует журнал событий Windows
[5] IOCTL Verification   → Проверяет активные IOCTL обработчики
[6] Handle Enumeration   → Проверяет подозрительные хэндлы
[7] Signature Check      → Проверяет тестовую подпись

РЕЗУЛЬТАТЫ
----------

[SECURE]     = Тест пройден, следов не найдено (ЗЕЛЕНЫЙ)
[VULNERABLE] = Тест провален, следы обнаружены (КРАСНЫЙ)

КРИТЕРИЙ УСПЕХА: 7/7 тестов должны пройти!

Если 7/7 → Безопасно использовать
Если < 7  → НЕ ИСПОЛЬЗОВАТЬ, исправить проблемы

ТИПИЧНЫЕ ПРОБЛЕМЫ
-----------------

Тест 1 провален → Переименуй все строки в маппере
Тест 2 провален → Очисти MmUnloadedDrivers в драйвере
Тест 3 провален → Очисти PiDDB cache в драйвере
Тест 4 провален → Очисти журнал событий после загрузки
Тест 5 провален → Измени IOCTL коды и имена устройств
Тест 6 провален → Закрой все хэндлы перед выгрузкой
Тест 7 провален → Отключи тестовую подпись (bcdedit /set testsigning off)

WORKFLOW
--------

1. Модифицируй KDMapper
   ↓
2. Собери модифицированный маппер
   ↓
3. Загрузи драйвер через маппер
   ↓
4. ЗАПУСТИ AUDITOR ← ТЫ ЗДЕСЬ
   ↓
5. Все тесты прошли? → Запускай клиент
   ↓
6. Тесты провалены? → Исправь и повтори с шага 1
